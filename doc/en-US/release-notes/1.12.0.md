# Chats 1.12.0 Release Notes

> Release Date: 2026-06-14 (51 non-merge commits since 1.11.0.671)

1.12.0 is centered on **model request customization** and **snapshot-based model configuration**. Admins can now configure custom request headers and request-body patches at both the **Model Key** and **Model** levels, and can override the request URL at the **Model** level to support more OpenAI-compatible gateways, proxy services, and provider-specific parameters. At the same time, Chats migrates model keys, models, and chat configs to immutable snapshots, so historical chats, usage, and billing are no longer affected by later model configuration changes. This release also fixes compatibility with the latest Claude Code by supporting Anthropic `role=system` messages, and includes improvements for image generation, Request Trace, image preview, model ordering, and multiple protocol integrations.

## 🎯 Core Features

### 1) Custom request headers and bodies for Model Key / Model

- 🧩 **Two-layer custom headers**: both Model Key and Model now support `Custom Headers` in one `Header-Name: value` pair per line. If the same header appears in both layers, the Model-level value wins, making it easy to add or override gateway parameters for a single model while sharing one key.
- 🧬 **Custom body patches**: both Model Key and Model now support `Custom Body` as an RFC 6902 JSON Patch array. It can apply `add`, `replace`, and similar operations to the final upstream request body, for example to add provider-specific parameters, disable a particular thinking mode, or adjust default request fields.
- 🔀 **Clear merge order**: request-body patches are applied first from Model Key, then from Model. If both layers touch the same path, the later Model patch naturally overrides the Key-level result.
- 🛡️ **Backend validation**: admin APIs validate header-line format and JSON Patch format so invalid overrides are rejected before real model calls fail.

### 2) Custom URL per Model

- 🌐 **Model-level URL override**: Model now has `Custom URL`, allowing one model to use a different upstream endpoint without duplicating or splitting the whole Model Key configuration.
- 🧱 **`{baseUrl}` placeholder**: custom URLs can reference the resolved Model Key base URL, which is useful when only the path or API version needs to change.
- 🔌 **Main request paths connected**: OpenAI Chat Completions, Responses, image generation/editing now use custom URL/Header/Body overrides; Anthropic Messages also supports Model-level URL override.

### 3) Snapshot-based model configuration: historical calls use historical facts

- 🧾 **New snapshot tables**: adds `ModelKeySnapshot`, `ModelSnapshot`, and `ChatConfigSnapshot`, splitting mutable head tables from versioned historical configuration.
- 🔒 **Stable historical facts**: `ChatTurn` now references `ChatConfigSnapshot`, while `UsageTransaction` and `UserModelUsage` reference `ModelSnapshot`. Later changes to model price, key, URL, or request overrides no longer rewrite the meaning of old usage and billing rows.
- 🗑️ **More controlled model deletion**: model and Model Key deletion logic is rewritten around the current snapshot relationship. Models still referenced by chat configs are disabled; unreferenced models can clean up user-model, cache, and API key associations before hard deletion.
- 🧭 **Model order fix**: the chat-side model list now follows the same provider, Model Key, and model order as the admin configuration.

## 🛠️ Protocol & Model Capabilities

### 1) Latest Claude Code compatibility

- 🤖 **Anthropic `role=system` messages**: Neutral Message now has a `System` role, and Anthropic Messages conversion preserves in-message `role=system`, making Chats compatible with the latest Claude Code request structure.
- 🧠 **API thinking blocks preserved**: API entry points no longer strip Anthropic `thinking` / `redacted_thinking` blocks simply because the request source is API. WebChat historical thinking trimming is handled in the chat context pipeline instead.
- 💳 **Anthropic cached token accounting fix**: fixes Anthropic usage handling so cached read tokens are included with input tokens instead of reading only `input_tokens`.

### 2) Reasoning levels and image generation parameters

- 🎚️ **String-based reasoning effort**: numeric reasoning effort values are migrated to strings: `minimal`, `low`, `medium`, `high`, `xhigh`, and `max`, making provider-specific extended levels easier to support.
- 🖼️ **Image output format**: image-generation models now support `Supported Output Formats`, and chat configs can choose formats such as `png`, `jpeg`, and `webp`.
- 📉 **Image output compression**: image generation adds `Output Compression` from 0 to 100, supported by both normal generation and multipart image editing requests.
- 🧪 **Model validation fix**: fixes a null-reference issue in the `ValidateModel` API after the large model configuration refactor.

## 🎨 UX Improvements

- 🔎 **Better model picker search**: the chat model dropdown can search by model name and provider, with improved mobile second-level model-selection animation.
- 🖼️ **Improved image preview**: image preview now supports better zoom, drag, and mobile pinch interactions; it also fixes zoom clipping and restores cross-`ResponseMessage` gallery navigation.
- 🧭 **ChatMiniMap and message layout**: single-column chat layout is centered on desktop, ChatMiniMap better distinguishes user messages, and Markdown tables no longer stretch response message layout.
- 📋 **Code block actions**: code blocks now support copy and expand/collapse actions, making long code easier to read and copy.
- 🪟 **Request Trace detail window**: request trace details now use a resizable Monaco-powered window, improving inspection of long headers, bodies, and dump content.
- 🧹 **Cleaner single-step turns**: single-step turns hide redundant per-step copy and info controls while preserving step edit actions.

## 🐛 Stability Fixes

- Fixes stale signed file URLs in message ETag caching.
- Fixes chat model list ordering so it matches the admin model order.
- Fixes image preview clipping after zoom.
- Fixes cross-`ResponseMessage` image gallery navigation in the Chat page.
- Fixes Anthropic model override URL loading behavior.
- Removes unused DeepSeek thinking handling and consolidates MiMo default thinking behavior and tests.
- Consolidates and adds unit tests for request overrides, Anthropic system role, reasoning effort normalization, and config snapshots.
- Updates Playwright to 1.60.0 and adds `pngquant` for screenshot/image processing workflows.

## ⬆️ Upgrade Notes

- Run DB migration: `src/scripts/db-migration/1.12/1.12.0.sql`
  - Adds `ModelKeySnapshot`, `ModelSnapshot`, and `ChatConfigSnapshot`
  - Adds `CurrentSnapshotId` to `ModelKey` and `Model`
  - Adds `ChatConfigSnapshotId` to `ChatTurn`
  - Adds `ModelSnapshotId` to `UsageTransaction` and `UserModelUsage`
  - Adds `SupportedFormats`, `OverrideUrl`, `CustomHeaders`, and `CustomBody` to `ModelSnapshot`
  - Adds `Format` and `Compression` to `ChatConfig` / `ChatConfigSnapshot`, and normalizes reasoning effort to strings
- The migration removes old mutable configuration columns from `Model` / `ModelKey`, removes `ChatConfigArchived`, and removes the old direct references `ChatTurn.ChatConfigId`, `UsageTransaction.ModelId`, and `UserModelUsage.ModelId`. If you have custom SQL, reports, or integrations that read those columns directly, update them to read the snapshot tables.
- `Custom Body` is not a full request-body replacement. It is a JSON Patch array; use a shape like `[{ "op": "replace", "path": "/temperature", "value": 0.7 }]`.
- Custom Header/Body values become part of the model snapshot history. After editing Model Key or Model configuration, new requests use the new snapshot, while old chats and billing records still resolve through the snapshot that existed at the time.

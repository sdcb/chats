# Chats 1.11.0 Release Notes

> Release Date: 2026-04-07 (50 commits since main)

1.11.0 is a release focused on **AI chat title summaries**, **model pricing transparency**, and **frontend interaction and performance polish**. It adds **first-turn AI-generated titles**, **admin and user layered title-summary configuration**, and a new **model pricing page**. It also breaks usage down by **WebChat / Api / Summary**, exposing both user-side filtering and admin-side source statistics. In addition, 1.11.0 adds **Novita AI** as the **22nd** model provider and continues improving desktop sidebar behavior, admin table interactions, Markdown rendering performance, and cross-provider protocol compatibility.

## 🎯 Core Features

### 1) Smart chat title summary: first-turn AI naming is now built in

- 🧠 **First-turn auto title**: after the first user message, Chats can asynchronously generate a more natural conversation title instead of relying only on raw text truncation; if summary generation fails, it falls back to the first 50 characters.
- 🧑‍💼 **Layered configuration**: admins can define global title-summary defaults, while users can save their own overrides in settings to control enablement, model mode, and prompt template.
- 🎛️ **Three model modes**: supports `truncate`, `current`, and `specified`, so title generation can reuse the current chat model or a separately designated model and prompt template.
- 💳 **Dedicated usage accounting**: title-summary requests are billed under the separate `Summary` source, making them easier to observe and cost-analyze apart from standard web chat and API usage.

### 2) Model pricing page: pricing, context windows, and free models in one place

- 💵 **New `/model-prices` page**: adds a dedicated page that lists Fresh Input, Cached Input, Output pricing, and context window information for available models.
- 🏷️ **Multi-dimensional filters**: supports filtering by model provider, API type, free-only models, and keyword search, making cost comparison faster in multi-model environments.
- 📚 **Better pricing clarity**: distinguishes Chat, Responses, Anthropic Messages, image generation, and other API styles so pricing is easier to understand.
- 📱 **Responsive layout**: the pricing page is tuned for both desktop and mobile with improved filter layout, scrolling behavior, and information density.

### 3) Usage source breakdown: WebChat / Api / Summary are now tracked separately

- 📊 **Persistent source field**: `UserModelUsage` now stores `SourceId`, classifying usage into `WebChat`, `Api`, and `Summary`.
- 🔎 **User-side filtering**: usage pages can now filter and export by source, helping users separate browser chat usage, OpenAI-compatible API usage, and title-summary overhead.
- 🧮 **Admin-side statistics**: admin statistics endpoints now expose source-based aggregation so different traffic entry points can be analyzed directly.
- 🔗 **Connected to title summaries**: because title summaries are billed under `Summary`, auto-title generation no longer gets mixed into normal chat statistics.

## 🏗️ Architecture & Engineering

### 1) User config storage and title-summary execution chain

- 🗄️ **UserConfig persistence**: adds a new `UserConfig` table keyed by `(UserId, Key)` to store user-level string configuration, providing a unified place for title-summary preferences and similar settings.
- ⚙️ **Global defaults + user override**: title-summary configuration now resolves from admin defaults plus per-user overrides through a shared resolution path.
- 🔁 **Shared execution path**: title generation now reuses the `ChatRunService` execution and billing pipeline so model selection, invocation, and usage recording stay aligned with the main chat flow.

### 2) Database structure and usage-statistics cleanup

- 🧹 **Removes the `TransactionType` lookup table**: transaction semantics no longer depend on a standalone database lookup table and are represented directly by code-level enums instead.
- 🧾 **Historical backfill**: the migration script backfills `SourceId` for existing `UserModelUsage` records, marking rows linked to `UserApiUsage` as `Api` and other legacy rows as `WebChat`.
- 🔒 **Tighter schema**: after backfill, `UserModelUsage.SourceId` is tightened to `NOT NULL` to avoid future source-less records in reporting.

### 3) Frontend performance and admin interaction consistency

- ⚡ **Lazy-loads heavy frontend resources**: Markdown rendering, KaTeX assets, charts, and parts of the chat session UI are now loaded on demand, cutting initial chat payload from roughly 3.5MB/3.6MB down to about 1.7MB.
- ⏱️ **Measurable load-time gains**: plain-text scenarios improved from about 1.1 minutes to 34.95 seconds; code-heavy scenarios reach first render in 33.37 seconds, with highlighted code content fully loaded at around 49.77 seconds.
- 🧱 **Progressive code block rendering**: before the syntax highlighter is ready, code blocks render as plain monospace text to reduce long blank waits for large code messages.
- 🗂️ **Unified admin tables**: user management, user models, and similar paginated tables are moving toward the request-trace interaction style, with debounced text filters to reduce jitter and over-refreshing.

## 🎨 Other Improvements

### Chat and interface experience

- ↔️ **Resizable desktop sidebar**: the desktop chat sidebar can now be resized by drag and keeps its width across later sessions.
- 💬 **Message bubble refinements**: mobile user bubbles fit short text more naturally, and collapsed sidebar content leakage is fixed.
- 📋 **More consistent copy behavior**: text extraction for copying is now centralized so both normal text messages and error messages copy more reliably.

### Model and protocol compatibility

- 🛠️ **More complete tool-response preservation**: continues fixing losses of tool outputs, tool-response images, and Anthropic usage fields across conversion and streaming-delta paths.
- 🆕 **Novita AI provider**: adds Novita AI as the 22nd model provider and supports reasoning information round-tripping.
- 🔧 **Provider compatibility refinements**: adds Gemini tool schema normalization and fixes Azure AI Foundry Grok `total_tokens` accounting.
- 🖼️ **More controlled Code Interpreter image flow**: images generated by code interpreter are no longer auto-converted into links and are no longer re-sent to upstream models by default; they are only attached for visual inspection when the AI explicitly calls the `view_image` tool.

## 🐛 Stability Fixes

- Fixes flicker caused by lazy-loaded Markdown components during streaming output.
- Fixes request failures caused by too-small `max_output_tokens` values by automatically raising them to a valid lower bound.
- Fixes Code Interpreter working-directory path resolution.
- Fixes vulnerable NuGet dependencies and clears related nullable build warnings.
- Fixes a set of scrollbar styles, dark-theme icon fill colors, and chat layout edge cases.

## ⬆️ Upgrade Notes

- Run DB migration: `src/scripts/db-migration/1.11/1.11.0.sql`
  - Removes the `TransactionType` table and related foreign keys
  - Adds `SourceId` to `UserModelUsage`, backfills historical data, then tightens it to `NOT NULL`
  - Adds the `UserConfig` table for user-level string configuration persistence
- If you have custom SQL, reporting, or external integrations that directly depend on the `TransactionType` table, update them before upgrading.
- Upgrading does not automatically enable title summaries for all users; `Summary` source requests only start once the admin configures global `chatTitleSummary`, or a user saves a personal title-summary config.
- Title summaries can either reuse the current chat model or use a separate designated model. If you plan to enable them in production, verify model access, pricing, and prompt templates first.

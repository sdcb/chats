# Chats 1.11.0 发布说明

> 发布日期：2026-04-07（相对 main 以来 50 次提交）

1.11.0 是一次围绕 **会话标题智能摘要**、**模型价格透明化** 和 **前端交互与性能打磨** 的版本。新版本新增 **首轮 AI 自动命名**、**管理员/用户双层标题摘要配置**、**模型定价页面**，并把用量来源细分为 **WebChat / Api / Summary**，补齐用户侧筛选与管理端来源统计。同时，Chats 在 1.11 中新增了第 **22** 个模型提供商 **Novita AI**，并继续推进桌面聊天侧边栏、后台表格交互、Markdown 渲染性能和多家模型协议兼容性的稳定性改进。

## 🎯 核心功能

### 1) 会话标题智能摘要：首轮 AI 自动命名正式落地

- 🧠 **首轮自动命名**：聊天在首次用户输入后可异步生成更自然的标题，不再只依赖原始输入截断；如果摘要失败或不可用，系统会自动回退到首 50 个字符作为兜底标题。
- 🧑‍💼 **双层配置**：管理员可在全局配置中设置标题摘要规则，普通用户也可在设置页创建个人 override，分别控制是否启用、使用哪种模型策略和提示词模板。
- 🎛️ **三种模型模式**：支持 `truncate`、`current`、`specified` 三种模式，可直接复用当前聊天模型，也可以为标题摘要单独指定模型与 Prompt Template。
- 💳 **独立用量归类**：标题摘要请求会以 `Summary` 来源单独记账，便于和普通 Web 聊天或 API 调用分开观察与核算。

### 2) 模型定价页面：价格、上下文窗口与免费模型一页看清

- 💵 **新增模型定价页**：新增 `/model-prices` 页面，集中展示当前可用模型的 Fresh Input、Cached Input、Output 单价与上下文窗口信息。
- 🏷️ **多维筛选**：支持按模型提供商、API 类型、免费模型和关键字搜索过滤结果，适合在多模型环境下快速找出合适的成本方案。
- 📚 **价格透明化**：页面把聊天、Responses、Anthropic Messages、图片生成等不同 API 类型明确区分，减少用户在选型和计费理解上的歧义。
- 📱 **响应式体验**：模型价格页针对桌面与移动端分别打磨了筛选布局、滚动行为和信息密度，避免大表格在小屏上难以浏览。

### 3) 用量来源细分：WebChat / Api / Summary 全部分开统计

- 📊 **来源字段持久化**：`UserModelUsage` 新增 `SourceId` 字段，把模型调用细分为 `WebChat`、`Api` 和 `Summary` 三类来源。
- 🔎 **用户侧筛选**：用量记录页现在支持按来源筛选与导出，用户可以更容易区分网页聊天消耗、开放 API 消耗和标题摘要带来的额外开销。
- 🧮 **管理端统计**：管理员统计接口新增按来源聚合的数据视角，可直接观察不同调用入口的占比变化。
- 🔗 **与标题摘要打通**：标题摘要使用 `Summary` 来源记账后，自动命名带来的调用量不再混入普通聊天统计，更便于成本分析与策略调优。

## 🏗️ 架构与工程

### 1) 用户配置与标题摘要链路补齐

- 🗄️ **用户配置落库**：新增 `UserConfig` 表，以 `(UserId, Key)` 为主键持久化用户级字符串配置，为标题摘要等个人偏好提供统一承载位置。
- ⚙️ **全局配置 + 个人覆盖**：标题摘要配置同时支持管理员全局默认值和用户个人覆盖，并通过统一的解析逻辑得到最终生效配置。
- 🔁 **复用统一执行链路**：标题摘要生成改为复用 `ChatRunService` 执行与计费链路，使标题生成、模型选择和用量记录逻辑与主聊天流程保持一致。

### 2) 数据库结构与统计口径整理

- 🧹 **移除 `TransactionType` 查找表**：交易类型不再依赖单独的数据库查找表，相关语义改为由代码中的枚举值直接承载。
- 🧾 **历史数据回填**：迁移脚本会为旧的 `UserModelUsage` 数据回填 `SourceId`，把存在 `UserApiUsage` 关联的历史记录标为 `Api`，其余旧记录标为 `WebChat`。
- 🔒 **字段收紧**：`UserModelUsage.SourceId` 在回填完成后被收紧为 `NOT NULL`，避免后续统计出现来源为空的脏数据。

### 3) 前端性能与后台交互一致性

- ⚡ **按需加载前端重资源**：Markdown 渲染、KaTeX 资源、图表和部分会话 UI 改为延迟加载，聊天页初始负载从约 3.5MB/3.6MB 降到约 1.7MB。
- ⏱️ **加载速度明显提升**：纯文本场景的加载时间从约 1.1 分钟降到 34.95 秒；带代码场景的首屏加载时间降到 33.37 秒，完整代码内容约在 49.77 秒完成加载。
- 🗂️ **后台表格统一化**：用户管理、用户模型等分页表格逐步统一为 request-trace 风格的交互方式，并为文本筛选补上防抖，减少频繁刷新与输入抖动。

## 🎨 其它改进

### 聊天与界面体验

- ↔️ **桌面侧边栏可调宽**：桌面聊天侧边栏支持拖拽调整宽度，并在后续会话中保持上次的宽度设置。
- 💬 **消息气泡细节优化**：移动端用户消息气泡更贴合短文本内容展示，同时修复折叠状态下侧边栏内容泄露的问题。
- 📋 **复制文本更一致**：消息复制逻辑抽出统一的可复制文本提取函数，文本消息和错误消息都能更稳定地复制出符合预期的内容。

### 模型与协议兼容性

- 🛠️ **Tool Response 保留更完整**：继续修复多家模型协议下 tool outputs、tool response 图片和 Anthropic usage 字段在转换或流式增量阶段的丢失问题。
- 🆕 **新增 Novita AI 提供商**：Chats 现已接入第 22 个模型提供商 Novita AI，并支持思考信息回传。
- 🔧 **提供商细节兼容**：补充 Gemini 工具 schema 归一化，以及 Azure AI Foundry Grok `total_tokens` 统计修正。
- 🖼️ **Code Interpreter 图片回传更可控**：AI 通过 code interpreter 生成的图片不再自动转成链接，也不会默认重新发送给上游模型；只有 AI 显式调用 `view_image` 工具时，图片才会作为视觉输入附加给模型继续观察。

## 🐛 修复与稳定性

- 修复懒加载 Markdown 组件在流式输出期间产生的闪烁问题。
- 修复 `max_output_tokens` 过小导致的请求失败，低于阈值时会自动提升到合法下限。
- 修复 Code Interpreter 工作目录路径解析问题。
- 修复存在已知漏洞的 NuGet 依赖，并清理相关可空构建警告。
- 修复若干滚动条样式、暗色图标填充色和聊天消息布局细节问题。

## ⬆️ 升级提示

- 运行数据库迁移脚本：`src/scripts/db-migration/1.11/1.11.0.sql`
  - 删除 `TransactionType` 表及其相关外键
  - 为 `UserModelUsage` 新增 `SourceId`，回填历史数据后收紧为 `NOT NULL`
  - 新增 `UserConfig` 表，用于持久化用户级字符串配置
- 如果你有直接依赖 `TransactionType` 表的自定义 SQL、报表或外部集成，升级前需要同步改造这些查询逻辑。
- 升级不会自动为所有用户开启标题摘要；只有在管理员配置了全局 `chatTitleSummary`，或用户自行保存了个人标题摘要配置后，系统才会开始发起 `Summary` 来源的标题生成调用。
- 标题摘要可以使用当前聊天模型，也可以单独指定模型；如果准备在生产环境开启该能力，建议先确认模型权限、价格和 Prompt Template，再决定是否全局开启。
